Create `aruba-leren/src/lib/children/actions.ts`:

1. **addChild(formData: FormData)** server action:
   - Mark 'use server'
   - Extract: firstName, age (number), grade (number 1-6) from formData
   - Validate: firstName non-empty, age 6-12, grade 1-6
   - Create server client, get authenticated user
   - If no user, return { error: 'Niet ingelogd' }
   - Get parent profile: `SELECT id FROM profiles WHERE user_id = user.id`
   - Insert into children table: { parent_id: profile.id, first_name: firstName, age, grade }
   - Return { error } or { success: true }
   - Use revalidatePath('/[locale]/dashboard') to refresh UI

2. **updateChild(childId: string, formData: FormData)** server action:
   - Extract: firstName, age, grade
   - Validate ownership: child belongs to current user's profile (via parent_id FK)
   - Update children table WHERE id = childId
   - Return { error } or { success: true }
   - Revalidate dashboard path

3. **deleteChild(childId: string)** server action:
   - Validate ownership
   - DELETE FROM children WHERE id = childId
   - Return { success: true }
   - Revalidate dashboard path

Create `aruba-leren/src/components/AddChildForm.tsx` (Client Component):

1. Import addChild action
2. Use useActionState or useState to handle form submission
3. Form fields:
   - Voornaam (text input, required)
   - Leeftijd (number input, min=6, max=12, required)
   - Klas (select dropdown with options 1-6, required)
4. Submit button: t('child.addChild')
5. Display success/error messages
6. Reset form on success
7. Styling: Card layout, sky-blue theme, kid-friendly

Create `aruba-leren/src/components/ChildList.tsx` (Client Component):

1. Accept `children` prop: Array<{ id: string, first_name: string, age: number, grade: number }>
2. If empty, show "Nog geen kinderen toegevoegd"
3. Map over children, render each as card with:
   - Name, age, grade (klas)
   - Edit button (opens inline edit form or modal)
   - Delete button (with confirmation)
4. Edit mode: Replace card content with editable inputs, call updateChild on save
5. Delete: Call deleteChild action with confirmation prompt
6. Styling: Grid layout on desktop, stack on mobile

Create `aruba-leren/src/app/[locale]/dashboard/page.tsx` (Server Component):

1. Accept `params: Promise<{ locale: string }>`
2. Create server client, call `getUser()`
3. If no user, redirect to `/[locale]/login` using redirect() from 'next/navigation'
4. Fetch current user's profile: `SELECT id FROM profiles WHERE user_id = user.id`
5. Fetch children: `SELECT * FROM children WHERE parent_id = profile.id ORDER BY created_at ASC`
6. Use getTranslations to load 'parent' and 'child' namespaces
7. Render:
   - Dashboard title: t('parent.dashboardTitle')
   - AddChildForm component
   - ChildList component with fetched children
   - Logout button (form with action={signOut} from '@/lib/auth/actions')
8. Layout: Max-width container, padding, sky-blue header

Translation keys (exist in nl.json):
