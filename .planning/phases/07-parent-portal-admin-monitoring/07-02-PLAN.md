**Migration 009** — Create `school_vacations` table:
```sql
CREATE TABLE IF NOT EXISTS school_vacations (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,
  start_date DATE NOT NULL,
  end_date DATE NOT NULL,
  school_year TEXT NOT NULL,
  is_public_holiday BOOLEAN NOT NULL DEFAULT false,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  CONSTRAINT valid_date_range CHECK (end_date >= start_date)
);
CREATE INDEX IF NOT EXISTS idx_school_vacations_year ON school_vacations(school_year, start_date ASC);
ALTER TABLE school_vacations ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Authenticated users can view school vacations" ON school_vacations FOR SELECT USING (auth.role() = 'authenticated');
CREATE TRIGGER update_school_vacations_updated_at BEFORE UPDATE ON school_vacations FOR EACH ROW EXECUTE FUNCTION update_updated_at_column();
```

**queries.ts** — Export `getVacations(schoolYear: string)` function:
- Uses regular Supabase server client (RLS allows authenticated read)
- Queries `school_vacations` filtered by `school_year`, ordered by `start_date` ascending
- Export a helper `getCurrentSchoolYear()`: Aruba school year starts August. If month >= 7 (August = index 7), year is `${currentYear}-${currentYear+1}`, else `${currentYear-1}-${currentYear}`
- Return typed array

**actions.ts** — Server Actions with `'use server'` directive:
- `upsertVacation(formData: FormData)`: Extract id (nullable), name, start_date, end_date, school_year, is_public_holiday from FormData. Check `isAdmin()` — return `{ error: 'Geen toegang' }` if not admin. Use `createAdminClient()` for insert (no id) or update (has id). Call `revalidatePath('/', 'layout')` — this invalidates all routes from the root layout, covering all locales (/nl/, /pap/, /es/, /en/) in one call. Return `{ success: true }`.
- `deleteVacation(id: string)`: Check `isAdmin()`. Use `createAdminClient()` to delete by id. Call `revalidatePath('/', 'layout')`. Return `{ success: true }`.
- Validate: name non-empty, start_date and end_date valid dates, end_date >= start_date. Return `{ error: string }` on validation failure.
  </action>
