  <action>
    Create `aruba-leren/supabase/migrations/20260213_initial_schema.sql` with the initial database schema.

    **Tables to create:**

    1. `profiles` table (parent accounts, linked to auth.users):
       - `id` UUID PRIMARY KEY DEFAULT gen_random_uuid()
       - `user_id` UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE (unique -- one profile per auth user)
       - `display_name` TEXT (parent's display name)
       - `locale` TEXT DEFAULT 'nl' CHECK (locale IN ('nl', 'pap', 'es')) -- stores language preference per locked decision
       - `created_at` TIMESTAMPTZ DEFAULT NOW()
       - `updated_at` TIMESTAMPTZ DEFAULT NOW()
       - UNIQUE constraint on user_id

    2. `children` table (child profiles, minimal data per PRIV-01):
       - `id` UUID PRIMARY KEY DEFAULT gen_random_uuid()
       - `parent_id` UUID NOT NULL REFERENCES profiles(id) ON DELETE CASCADE
       - `first_name` TEXT NOT NULL -- voornaam only, per PRIV-01
       - `age` INTEGER CHECK (age >= 4 AND age <= 14)
       - `grade` INTEGER NOT NULL CHECK (grade >= 1 AND grade <= 6) -- klas 1-6
       - `created_at` TIMESTAMPTZ DEFAULT NOW()
       - `updated_at` TIMESTAMPTZ DEFAULT NOW()

    **RLS for each table (CRITICAL - enable IMMEDIATELY after CREATE TABLE):**

    For `profiles`:
    - ALTER TABLE profiles ENABLE ROW LEVEL SECURITY;
    - CREATE INDEX idx_profiles_user_id ON profiles(user_id);
    - SELECT policy: authenticated users can view own profile (auth.uid() = user_id)
    - INSERT policy: authenticated users can create own profile (auth.uid() = user_id)
    - UPDATE policy: authenticated users can update own profile (auth.uid() = user_id)

    For `children`:
    - ALTER TABLE children ENABLE ROW LEVEL SECURITY;
    - CREATE INDEX idx_children_parent_id ON children(parent_id);
    - SELECT policy: users can view children belonging to their profile
      `USING (parent_id IN (SELECT id FROM profiles WHERE user_id = auth.uid()))`
    - INSERT policy: users can add children to their own profile
      `WITH CHECK (parent_id IN (SELECT id FROM profiles WHERE user_id = auth.uid()))`
    - UPDATE policy: users can update their own children
      `USING (parent_id IN (SELECT id FROM profiles WHERE user_id = auth.uid()))`
    - DELETE policy: users can remove their own children
      `USING (parent_id IN (SELECT id FROM profiles WHERE user_id = auth.uid()))`

    **Updated_at trigger:**
    Create a function `update_updated_at_column()` that sets `updated_at = NOW()` on UPDATE. Create triggers for both tables.

    **Comments in SQL:**
    - Document each table's purpose
    - Note that this stores ONLY the data required by PRIV-01 (voornaam, klas, voortgang)
    - Note that voortgangsdata tables will be added in Phase 5
    - Note RLS is deny-by-default: without policies, all queries return empty

    Do NOT create tables for: payments, sessions, chat messages, progress -- those belong to later phases.
