  <name>Task 1: Stel alle 6 environment variables in via Vercel Dashboard</name>
  <action>
    1. Ga naar https://vercel.com → jouw project (aruba-leren) → "Settings" → "Environment Variables"

    2. Voeg de volgende 6 variabelen toe. Stel voor elke variabele de scope in op "Production" (vink aan: Production; vink optioneel Preview + Development uit voor geheime keys):

    ---
    Variabele 1: NEXT_PUBLIC_SUPABASE_URL
    Waarde: Haal op via Supabase Dashboard → Project Settings → API → "Project URL"
    Voorbeeld: https://abcdefghijklmn.supabase.co
    Scope: Production (en optioneel Preview/Development)
    ---

    Variabele 2: NEXT_PUBLIC_SUPABASE_ANON_KEY
    Waarde: Haal op via Supabase Dashboard → Project Settings → API → "anon" key (public)
    Scope: Production (en optioneel Preview/Development)
    ---

    Variabele 3: SUPABASE_SERVICE_ROLE_KEY
    Waarde: Haal op via Supabase Dashboard → Project Settings → API → "service_role" key
    WAARSCHUWING: Dit is een geheime key. Selecteer ALLEEN scope "Production" (niet Preview/Development als je niet zeker bent).
    Scope: Production only
    ---

    Variabele 4: ANTHROPIC_API_KEY
    Waarde: Haal op via https://console.anthropic.com → API Keys
    Begint met: sk-ant-...
    Scope: Production only
    ---

    Variabele 5: OPENAI_API_KEY
    Waarde: Haal op via https://platform.openai.com → API keys
    Begint met: sk-...
    Scope: Production only
    ---

    Variabele 6: NEXT_PUBLIC_APP_URL
    Waarde: De exacte productie-URL uit Plan 01 (bijv. https://aruba-leren.vercel.app)
    LET OP: Geen trailing slash. Gebruik HTTPS. Gebruik de echte Vercel-URL, NIET localhost.
    Scope: Production (en optioneel Preview/Development)
    ---

    3. Klik na elke variabele op "Save" (of voeg ze allemaal toe en sla één keer op, afhankelijk van de Vercel-UI).

    4. Controleer de lijst: je ziet nu 6 variabelen in de "Production" omgeving.

    VEELGEMAAKTE FOUT: NEXT_PUBLIC_APP_URL vergeten of op localhost laten staan → rapport-deellinks werken niet.

<task type="checkpoint:human-action">
  <name>Task 2: Trigger een nieuwe deployment en verifieer dat API-routes werken</name>
  <action>
    BELANGRIJK: Env var wijzigingen gelden NIET voor de bestaande deployment. Je moet een nieuwe deployment triggeren.

    OPTIE A — Via Vercel Dashboard (aanbevolen als je geen commit wilt pushen):
    1. Ga naar Vercel Dashboard → jouw project → "Deployments"
    2. Klik op de meest recente deployment
    3. Klik op de "..." menu (drie puntjes, rechtsboven bij de deployment)
    4. Kies "Redeploy" → bevestig in het popup-venster
    5. Wacht tot de nieuwe deployment de status "Ready" heeft

    OPTIE B — Via git push:
    1. Maak een kleine lege commit: git commit --allow-empty -m "chore: trigger production redeploy with env vars"
    2. Push naar de main branch: git push origin main
    3. Vercel detecteert de push automatisch en start een nieuwe deployment

    Na de nieuwe deployment (status "Ready"):
    4. Open de productie-URL in de browser
    5. Navigeer naar de inlogpagina — de pagina laadt correct (geen "Internal Server Error")
    6. Optioneel snelle API-test: open de browser-console (F12) en controleer of er geen 500-errors zijn bij het laden van de pagina
  </action>
  <verify>
    - Vercel Dashboard → Deployments: de nieuwste deployment heeft status "Ready" en een tijdstempel NA het instellen van de env vars
    - De productie-URL is bereikbaar in de browser
    - De landingspagina en inlogpagina laden zonder "Application error" of "Internal Server Error"
  </verify>
  <done>Nieuwe deployment met env vars is live. De app laadt correct. API-routes zijn niet meer gebroken door ontbrekende configuratie.</done>
</task>
