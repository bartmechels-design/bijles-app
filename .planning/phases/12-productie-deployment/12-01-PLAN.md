<task type="checkpoint:human-action">
  <name>Task 1: Importeer repo in Vercel met Root Directory aruba-leren</name>
  <action>
    KRITIEKE INSTELLING: Root Directory MOET "aruba-leren" zijn — dit is de meest voorkomende oorzaak van een mislukte eerste build.

    Stap-voor-stap:
    1. Ga naar https://vercel.com en log in op je account
    2. Klik op "Add New..." → "Project"
    3. Selecteer "Import Git Repository" en kies de GitHub-repository "Bijles App" (of de juiste naam)
       - Als je repo nog niet gekoppeld is: klik "Adjust GitHub App Permissions" en verleen toegang
    4. VOORDAT je op Deploy klikt: zoek het veld "Root Directory" (staat soms in een uitklapbaar gedeelte "Build and Output Settings")
    5. Stel Root Directory in op: aruba-leren
       (Dit is de map die package.json en next.config.ts bevat — niet de repo-root)
    6. Controleer dat Framework Preset automatisch op "Next.js" wordt gezet na het invullen van de Root Directory
    7. Laat Build Command, Install Command en Output Directory op de standaard staan (niet aanpassen)
    8. Klik "Deploy" — de eerste build start nu

    WAARSCHUWINGSSIGNALEN: Als je in de build-logs ziet "No Next.js version detected" of "Cannot find package.json" → Root Directory is fout ingesteld. Stop de build, ga naar Project Settings → General → Root Directory en corrigeer naar "aruba-leren". Klik daarna "Redeploy".

    Na succesvolle build: noteer de productie-URL (bijv. https://aruba-leren.vercel.app) — je hebt deze nodig in Plan 02, 03 en 04.
  </action>
  <verify>
    - Vercel Dashboard → Deployments toont de nieuwste deployment met status "Ready" (groen vinkje)
    - Bezoek de productie-URL in de browser: de landingspagina laadt (mag tijdelijk 500-errors geven op API-routes door ontbrekende env vars — dat is normaal en wordt opgelost in Plan 02)
    - De URL begint met https://
  </verify>
  <done>Vercel-project bestaat, eerste build is geslaagd (Ready), productie-URL is bekend en genoteerd.</done>
</task>
