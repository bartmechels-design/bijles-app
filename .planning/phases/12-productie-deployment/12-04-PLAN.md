<objective>
Configureer Google OAuth en Facebook OAuth met de productie callback-URL en stel de Supabase Site URL in, zodat sociale logins werken op productie.

Purpose: Zonder deze configuratie stuurt OAuth-login gebruikers terug naar localhost of geeft een "redirect_uri_mismatch" fout. Dit is de laatste stap vóór de app volledig functioneel is voor eindgebruikers.
Output: Google Login en Facebook Login werken op de productie-URL en leiden correct terug naar de app. De Facebook-app staat in Live-modus zodat niet-admin gebruikers kunnen inloggen.

Vereisten: Plan 01 (productie-URL bekend) en Plan 03 (env vars geconfigureerd) moeten klaar zijn.
</objective>

<task type="checkpoint:human-action">
  <name>Task 1: Update Supabase URL Configuration + Google OAuth</name>
  <action>
    STAP A: Supabase Site URL bijwerken (EERST doen — dit bepaalt waarheen OAuth redirect)

    1. Ga naar https://supabase.com/dashboard → jouw project
    2. Ga naar "Authentication" → "URL Configuration"
    3. Stel "Site URL" in op de productie-URL uit Plan 01
       Voorbeeld: https://aruba-leren.vercel.app
       (Verwijder de oude waarde http://localhost:3000)
    4. Voeg onder "Redirect URLs" de volgende URL toe (klik op "+ Add URL"):
       https://aruba-leren.vercel.app/**
       (Gebruik ** voor wildcard — dit staat alle sub-paths toe)
    5. Houd http://localhost:3000/** in de lijst staan voor lokale ontwikkeling
    6. Klik "Save"

    STAP B: Zoek de Supabase OAuth Callback URL

    7. Ga naar Supabase Dashboard → "Authentication" → "Providers" → klik op "Google"
    8. Kopieer de "Callback URL (for OAuth)" — dit is iets als:
       https://abcdefghijklmn.supabase.co/auth/v1/callback
       LET OP: Dit is NIET de app-URL. Dit is de Supabase server die de OAuth afhandelt.
    9. Bewaar deze URL — je hebt hem nodig in de Google Cloud Console.

    STAP C: Google Cloud Console configureren

    10. Ga naar https://console.cloud.google.com
    11. Selecteer het juiste project (het project waarmee de Google OAuth voor deze app is aangemaakt)
    12. Ga naar "APIs & Services" → "Credentials"
    13. Klik op de OAuth 2.0 Client ID die voor deze app gebruikt wordt
    14. Onder "Authorized JavaScript origins": klik "ADD URI" en voeg toe:
        https://aruba-leren.vercel.app   (de productie-URL uit Plan 01)
    15. Onder "Authorized redirect URIs": klik "ADD URI" en voeg toe:
        https://abcdefghijklmn.supabase.co/auth/v1/callback   (de callback URL uit stap 8)
    16. Klik "SAVE"

    STAP D: Supabase Google provider activeren (als nog niet gedaan)

    17. Terug in Supabase Dashboard → Authentication → Providers → Google:
        - Zorg dat "Enable Sign in with Google" aangevinkt is
        - Voer de Client ID en Client Secret in vanuit de Google Cloud Console Credentials pagina
        - Klik "Save"

    VEELGEMAAKTE FOUT: De app-URL (aruba-leren.vercel.app) invullen als redirect URI in Google Console in plaats van de Supabase callback URL (*.supabase.co/auth/v1/callback). Google stuurt de OAuth-callback naar Supabase, niet direct naar de app.
  </action>
  <verify>
    Test Google OAuth:
    1. Ga naar de productie-URL (bijv. https://aruba-leren.vercel.app/nl/login)
    2. Klik op "Inloggen met Google"
    3. Google-loginscherm verschijnt
    4. Log in met een Google-account
    5. Na login: browser redirect gaat naar de productie-app (NIET naar localhost)
    6. Je bent ingelogd en ziet het dashboard

    Als je een "Error 400: redirect_uri_mismatch" ziet van Google:
    → De Supabase callback URL staat nog niet in de Authorized redirect URIs van de Google Console
    → Controleer stap 15 en wacht 5 minuten (Google-wijzigingen hebben soms propagatietijd)
  </verify>
  <done>Supabase Site URL is bijgewerkt naar productie. Google OAuth werkt op productie-URL en leidt correct terug naar de app.</done>
</task>

<task type="checkpoint:human-action">
  <name>Task 2: Facebook OAuth configureren en App Mode op Live zetten</name>
  <action>
    STAP A: Supabase Facebook callback URL ophalen

    1. Ga naar Supabase Dashboard → Authentication → Providers → Facebook
    2. Kopieer de "Callback URL (for OAuth)":
       https://abcdefghijklmn.supabase.co/auth/v1/callback
       (Zelfde URL als Google — de Supabase callback, niet de app-URL)

    STAP B: Facebook Developer Console — OAuth Redirect URI toevoegen

    3. Ga naar https://developers.facebook.com en log in
    4. Selecteer de juiste app (of maak een nieuwe aan als er nog geen Facebook-app bestaat voor dit project)
       Als je een nieuwe app aanmaakt: kies "Consumer" als app-type. Voeg "Facebook Login" toe als product.
    5. Ga naar jouw app → "Facebook Login" → "Settings" (in de linker sidebar)
    6. Zoek het veld "Valid OAuth Redirect URIs"
    7. Voeg toe: https://abcdefghijklmn.supabase.co/auth/v1/callback (de callback uit stap 2)
    8. Klik "Save Changes"

    STAP C: Privacy Policy URL instellen (VEREIST voor Live modus)

    9. Ga naar jouw Facebook app → "Settings" → "Basic" (in de linker sidebar)
    10. Vul in het veld "Privacy Policy URL" de URL van de privacy-pagina van de app in:
        https://aruba-leren.vercel.app/nl/privacy
        (De app heeft een privacy policy pagina op /[locale]/privacy)
    11. Klik "Save Changes"

    STAP D: Facebook App Mode omzetten naar Live

    12. Nog steeds in "Settings" → "Basic": zoek de schakelaar "App Mode" (staat bovenaan de pagina, toont "Development" of "In Development")
    13. Zet de schakelaar op "Live"
    14. Bevestig in het popup-venster (Facebook kan vragen of je de permissies bevestigt)

    LET OP: Als Facebook zegt "Your app cannot go live yet" — controleer of alle verplichte velden zijn ingevuld: Privacy Policy URL, App Icon, App Category. Voeg ontbrekende velden toe.

    STAP E: Supabase Facebook provider activeren

    15. Ga naar Supabase Dashboard → Authentication → Providers → Facebook
        - Zorg dat "Enable Sign in with Facebook" aangevinkt is
        - Voer "App ID" in (te vinden op Facebook Developer Console → Settings → Basic)
        - Voer "App Secret" in (klik op "Show" naast het veld in Facebook Console → Settings → Basic)
        - Klik "Save"
  </action>
  <verify>
    Test Facebook OAuth:
    1. Open een incognito/privé venster (belangrijk — sluit bestaande sessies uit)
    2. Ga naar https://aruba-leren.vercel.app/nl/login
    3. Klik op "Inloggen met Facebook"
    4. Facebook-loginscherm verschijnt
    5. Log in met een Facebook-account dat GEEN admin of test-gebruiker is van de Facebook Developer app
       (Dit bewijst dat de Live-modus correct werkt voor gewone gebruikers)
    6. Na login: browser redirect gaat naar de productie-app (NIET naar localhost)
    7. Je bent ingelogd

    Als je "App Not Setup" of "User Not Authorized" ziet:
    → De Facebook app staat nog in Development mode — controleer stap 12/13
    → Zorg dat de App Mode schakelaar op "Live" staat

    Als je "The redirect_uri is not allowed" ziet:
    → De Supabase callback URL staat niet in Valid OAuth Redirect URIs — controleer stap 7/8
  </verify>
  <done>Facebook OAuth werkt op productie-URL voor niet-admin gebruikers. Facebook app staat in Live modus. Inloggen via Facebook leidt correct terug naar de productie-app.</done>
</task>
