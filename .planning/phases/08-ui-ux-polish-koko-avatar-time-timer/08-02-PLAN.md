  <action>
Create a new `'use client'` component `TimeTimer` with the following API:

```ts
interface TimeTimerProps {
  duration: number          // Total duration in seconds
  elapsed: number           // Elapsed seconds (controlled from parent)
  size?: number             // Pixel size (default: 56)
  showDigits?: boolean      // Show MM:SS inside circle (default: false)
  soundEnabled?: boolean    // Enable completion bell (default: false)
  onComplete?: () => void   // Callback when time expires
}
```

**Note:** elapsed is passed from parent (SessionTimer already tracks elapsed state). TimeTimer is a pure visual component that does NOT own its own timer interval.

**SVG structure:**
1. SVG viewBox="0 0 100 100", width/height from `size` prop.
2. Background circle: cx=50, cy=50, r=45, stroke="#E5E7EB" (gray-200), strokeWidth=8, fill="none".
3. Progress circle: same center/radius, stroke="#EF4444" (red-500), strokeWidth=8, fill="none", strokeLinecap="round".
   - `strokeDasharray={circumference}` where circumference = 2 * Math.PI * 45 (approximately 282.74).
   - `strokeDashoffset={circumference * (elapsed / duration)}` — starts at 0 (full red circle), grows to circumference (empty) as elapsed increases.
   - Apply `style={{ transition: 'stroke-dashoffset 0.5s linear' }}` for smooth animation between second updates.
   - Apply `transform="rotate(-90 50 50)"` so arc starts at 12 o'clock position.
4. When remaining time < 20% of duration, pulse the red color by adding CSS class `time-timer-warning` which applies `animation: time-timer-pulse 1s ease-in-out infinite`.
5. If `showDigits` is true, render a `<text>` element centered (x=50, y=55, textAnchor="middle", dominantBaseline="middle") showing remaining time as `M:SS` format (not zero-padded minutes). Font size 18, fill="#374151" (gray-700), fontWeight bold.

**Sound (Web Audio API):**
1. Use a `useRef<AudioContext | null>` initialized to null.
2. Create AudioContext lazily: only when `soundEnabled` is true AND `onComplete` fires (or when elapsed >= duration).
3. `playBell()` function: create oscillator at 523Hz (C5), type sine, connect to gain node starting at 0.3, exponentialRampToValueAtTime(0.001, currentTime + 1.5). Start oscillator, stop after 1.5s. This is a gentle completion chime.
4. Use a `useRef<boolean>` (`bellPlayed`) to ensure bell only fires once per timer lifecycle. Reset when duration changes.
5. In a `useEffect`, when `elapsed >= duration && soundEnabled && !bellPlayed.current`, call playBell() and set bellPlayed to true. Also call `onComplete?.()`.

**CSS (inline or in component):**
- The warning pulse animation should be defined inline via a `<style>` tag inside the component (avoids globals.css conflict with plan 01):
  ```css
  @keyframes time-timer-pulse {
    0%, 100% { opacity: 1; }
    50% { opacity: 0.5; }
  }
  ```

**Constraints:**
- Do NOT use setInterval inside TimeTimer — it receives elapsed from parent.
- Component must be pure/controlled: same props = same render.
- AudioContext must be created inside a user interaction context (the parent SessionTimer's "continue" button click counts). If AudioContext creation fails (autoplay policy), silently catch and skip.
