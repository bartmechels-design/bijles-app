**admin/payments/actions.ts** (server actions with 'use server'):

`approvePaymentRequest(formData: FormData)`:
1. Check isAdmin(). Return error 'Niet geautoriseerd' if not admin.
2. Extract requestId from formData.
3. Create admin client via createAdminClient() (service role, bypasses RLS).
4. Fetch payment request by id with `.select('*, profiles(id)')`.
5. If not found or not pending, return error.
6. Calculate expiry date using calculateExpiryDate(request.subscription_period) from @/lib/subscription/check.
7. Get current admin user id for audit trail (via server Supabase client auth.getUser()).
8. Upsert into subscriptions: profile_id = request.profile_id, status = 'active', period = request.subscription_period, expires_at = calculated date, last_payment_request_id = requestId. Use onConflict: 'profile_id' to handle existing subscriptions (renewal).
9. Update payment_requests: status = 'approved', approved_at = new Date().toISOString(), approved_by_user_id = admin user id.
10. revalidatePath('/admin/payments').
11. Return { success: true }.

`rejectPaymentRequest(formData: FormData)`:
1. Check isAdmin().
2. Extract requestId and reason from formData.
3. Create admin client.
4. Update payment_requests: status = 'rejected', rejected_at = new Date().toISOString(), rejected_reason = reason.
5. revalidatePath('/admin/payments').
6. Return { success: true }.

- Uses client-side Supabase to check subscription status
- On mount (useEffect):
