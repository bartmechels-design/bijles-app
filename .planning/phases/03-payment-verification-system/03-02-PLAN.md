  <action>
Create server actions file with 'use server' directive.

**uploadPaymentProof(formData: FormData):**
1. Get authenticated user via server Supabase client. Return error if not authenticated.
2. Extract fields from formData: payment_proof (File), subscription_period (string), comment (string, optional).
3. Validate subscription_period is valid SubscriptionPeriod type (import from @/lib/subscription/types).
4. Call validateImageFile(file) from @/lib/storage/upload. Return translated error if invalid.
5. Upload file to Supabase Storage bucket 'payment-proofs' with path `{user.id}/{Date.now()}-{file.name}`. Use `file` directly (not buffer) in supabase.storage.from('payment-proofs').upload().
6. If upload error, return { error: 'Bestand kon niet worden geupload' }.
7. Get profile_id by querying profiles where user_id = user.id.
8. Insert into payment_requests: profile_id, subscription_period, payment_method: 'bank_transfer', payment_proof_path: filePath, status: 'pending', comment (if provided).
9. revalidatePath for subscription status page.
10. Return { success: true }.

**requestCashPayment(formData: FormData):**
1. Get authenticated user. Return error if not authenticated.
2. Extract: subscription_period (string), comment (string -- required for cash, describes when/where payment).
3. Validate subscription_period.
4. Validate comment is not empty (required for cash payments): return error 'Vul een toelichting in wanneer en waar u wilt betalen'.
5. Get profile_id.
6. Insert into payment_requests: profile_id, subscription_period, payment_method: 'cash', payment_proof_path: null, status: 'pending', comment.
7. revalidatePath.
8. Return { success: true }.

Both actions return `{ success?: boolean; error?: string }`.
