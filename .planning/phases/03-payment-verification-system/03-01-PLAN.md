Create two SQL migration files.

**004_subscriptions_payments.sql** must contain:

1. Create enums: `subscription_status` (active, expired, cancelled), `subscription_period` (per_session, per_week, per_month, per_school_year), `payment_method` (bank_transfer, cash), `payment_request_status` (pending, approved, rejected).

2. Create `subscriptions` table with columns:
   - id UUID PK default gen_random_uuid()
   - profile_id UUID NOT NULL REFERENCES profiles(id) ON DELETE CASCADE, UNIQUE constraint (one subscription per parent)
   - status subscription_status NOT NULL DEFAULT 'active'
   - period subscription_period NOT NULL
   - expires_at TIMESTAMPTZ NOT NULL
   - last_payment_request_id UUID (nullable, no FK constraint to avoid circular dependency)
   - created_at TIMESTAMPTZ DEFAULT NOW()
   - updated_at TIMESTAMPTZ DEFAULT NOW()

3. Create `payment_requests` table with columns:
   - id UUID PK default gen_random_uuid()
   - profile_id UUID NOT NULL REFERENCES profiles(id) ON DELETE CASCADE
   - subscription_period subscription_period NOT NULL
   - payment_method payment_method NOT NULL
   - payment_proof_path TEXT (NULL for cash payments)
   - status payment_request_status NOT NULL DEFAULT 'pending'
   - comment TEXT (parent note for cash payments)
   - approved_at TIMESTAMPTZ
   - approved_by_user_id UUID REFERENCES auth.users(id)
   - rejected_at TIMESTAMPTZ
   - rejected_reason TEXT
   - created_at TIMESTAMPTZ DEFAULT NOW()
   - updated_at TIMESTAMPTZ DEFAULT NOW()

4. Enable RLS on both tables.

5. Create performance indexes:
   - idx_subscriptions_profile_status ON subscriptions(profile_id, status, expires_at)
   - idx_subscriptions_expiry ON subscriptions(expires_at) WHERE status = 'active'
   - idx_payment_requests_profile ON payment_requests(profile_id)
   - idx_payment_requests_status ON payment_requests(status) WHERE status = 'pending'

6. Create RLS policies:
   - subscriptions: Users can SELECT own (profile_id matches via profiles.user_id = auth.uid())
   - subscriptions: Admin can SELECT all (auth.jwt() -> 'app_metadata' ->> 'role' = 'admin')
   - payment_requests: Users can SELECT own
   - payment_requests: Users can INSERT own
   - payment_requests: Admin can SELECT all
   - payment_requests: Admin can UPDATE all (for approve/reject)

7. Create `user_has_active_subscription()` function (SECURITY DEFINER, STABLE, LANGUAGE SQL) that checks if auth.uid() has an active subscription with expires_at > NOW(), joining through profiles table.

**005_storage_policies.sql** must contain:
- Comment noting that the `payment-proofs` private bucket must be created via Supabase Dashboard (Storage > New Bucket > name: payment-proofs, Public: off, File size limit: 5MB, Allowed MIME types: image/jpeg, image/png, image/webp)
- RLS policy: Users can INSERT to storage.objects where bucket_id = 'payment-proofs' AND folder name matches auth.uid()
- RLS policy: Users can SELECT from storage.objects where bucket_id = 'payment-proofs' AND folder name matches auth.uid()
- Use (storage.foldername(name))[1] = auth.uid()::text for folder matching

NOTE: These migrations are documentation/reference SQL. The actual execution happens via Supabase Dashboard SQL Editor since there is no local Supabase CLI setup. Include a comment at the top of each file: "-- Run this SQL in Supabase Dashboard > SQL Editor"

**src/lib/storage/upload.ts:**
- Import `fileTypeFromBuffer` from 'file-type'
- Export const ALLOWED_IMAGE_TYPES = ['image/jpeg', 'image/png', 'image/webp']
- Export const MAX_FILE_SIZE = 5 * 1024 * 1024 (5MB)
- Export async function `validateImageFile(file: File): Promise<{ valid: boolean; error?: string }>`:
  1. Check file.size > MAX_FILE_SIZE -> return error 'Bestand is te groot (max 5MB)'
  2. Convert file to ArrayBuffer, then to Buffer (use new Uint8Array for edge runtime compatibility)
  3. Call fileTypeFromBuffer on the buffer
  4. If no fileType detected -> return error 'Bestandstype kon niet worden bepaald'
  5. If fileType.mime not in ALLOWED_IMAGE_TYPES -> return error with detected type
  6. Return { valid: true }

NOTE on file-type: This is an ESM-only package. If import issues arise with Next.js, use dynamic import: `const { fileTypeFromBuffer } = await import('file-type')`. The function is async anyway so this works fine.
