       c. In system prompt building section: if isAssessment, use `buildAssessmentPrompt(subject, tutorLanguage, child.age, child.first_name, child.grade)` instead of `buildSystemPrompt(...)`. Skip difficulty adjustment for assessment sessions (Koko manages difficulty internally via the prompt).
       d. In onFinish callback, AFTER existing analysis and metadata update code, add three new blocks:

       BLOCK 1 — Assessment completion detection:
       ```
       if (isAssessment) {
         const levelMatch = text.match(/\[ASSESSMENT_DONE:level=([1-5])\]/);
         if (levelMatch) {
           const determinedLevel = parseInt(levelMatch[1]);
           try {
             await finishAssessment(childId, subject, determinedLevel, currentSession.id);
           } catch (e) {
             console.error('Assessment finalization failed:', e);
           }
         }
       }
       ```

       BLOCK 2 — Stuck detection (works for both assessment and tutoring):
       ```
       const currentConsecutiveIncorrect =
         metadataUpdates.consecutive_incorrect ?? currentSession.metadata.consecutive_incorrect;
       if (currentConsecutiveIncorrect >= 3 && analysis.wasCorrect !== true) {
         try { await updateStuckFlag(childId, subject, true); } catch (e) { console.error('Stuck flag failed:', e); }
       }
       if (analysis.wasCorrect === true) {
         try { await updateStuckFlag(childId, subject, false); } catch (e) { /* non-fatal */ }
       }
       ```

       BLOCK 3 — Level change tracking (tutoring sessions only):
       ```
       if (!isAssessment && difficultyAdjustment.reason !== 'no_change') {
         try {
           await recordProgressEvent(
             childId, subject, currentSession.id,
             difficultyAdjustment.newDifficulty > currentSession.difficulty_level ? 'level_up' : 'level_down',
             difficultyAdjustment.newDifficulty,
             difficultyAdjustment.reason
           );
         } catch (e) { console.error('Progress event failed:', e); }
       }
       ```

       e. IMPORTANT: The `isAssessment` variable must be accessible inside onFinish. Since onFinish is a closure, it will capture `currentSession` which has `session_type`. Compute `isAssessment` outside the closure or derive it inside from `currentSession.session_type`.
