Voeg onderaan globals.css een nieuwe sectie toe: `/* ===== Whiteboard schrijfanimatie ===== */`

Voeg deze CSS toe:

```css
/* ===== Whiteboard schrijfanimatie ===== */

@keyframes reveal-line {
  from {
    clip-path: inset(0 100% 0 0);
  }
  to {
    clip-path: inset(0 0% 0 0);
  }
}

.whiteboard-line {
  animation: reveal-line 0.6s cubic-bezier(0.25, 0.46, 0.45, 0.94) both;
}

@media (prefers-reduced-motion: reduce) {
  .whiteboard-line {
    animation: none;
  }
}
```

OPMERKING: `clip-path: inset(...)` is compositor-threaded op Android — gebruik NOOIT `stroke-dashoffset` (werkt niet betrouwbaar op low-end Android). Gebruik GEEN SVG-gebaseerde tekst-animaties hier.

Wijzig de render-logica in Whiteboard.tsx zodat elk blok dat voor het eerst zichtbaar wordt de CSS-klasse `whiteboard-line` krijgt met een stagger-delay.

**Hoe de huidige code werkt:**
- `visibleLines` telt het aantal zichtbare "lijnen" (1 per parsed line).
- `blockLineRange(blockIdx)` geeft `{ start, end }` voor elk blok.
- Een blok is zichtbaar wanneer `range.start < visibleLines`.
- Huidige animatie: `transition-opacity duration-300` op de container div.

**Wat te veranderen:**

1. Verwijder de klasse `transition-opacity duration-300` overal in de JSX (er zijn ~4 plekken waar dit voorkomt op zichtbare elementen).

2. Voeg voor elk blok een berekende `animationDelay` toe die bepaalt wanneer de schrijfanimatie start:
   ```
   const animDelay = `${range.start * 100}ms`
   ```
   (range.start = lineIndex waarop dit blok start — dit zorgt voor staggering)

3. Pas de className en style toe op de outer div/element van elk zichtbaar blok:
   - Voor sum-block items: op de `<div key={li}` (sum-number en sum-line):
     ```tsx
     className="flex whiteboard-line"
     style={{ width: `...`, animationDelay: `${(range.start + li) * 80}ms` }}
     ```
   - Voor label-blokken: op de `<div key={blockIdx}` container:
     ```tsx
     className="whiteboard-line"
     style={{ animationDelay: animDelay }}
     ```
   - Voor plain text `<p>`:
     ```tsx
     className="text-base text-slate-800 whiteboard-line"
     style={{ animationDelay: animDelay }}
     ```

4. sum-line (de horizontale streep) hoeft GEEN clip-path animatie — behoud de huidige stijl voor de streep zelf (borderBottom). De container div om sum-lines heen krijgt wel de whiteboard-line klasse.

**Belangrijke aandachtspunten:**
- Verwijder ALLEEN de `transition-opacity` classes — verwijder NIET andere logica.
- De `visibleLines` state en de setTimeout-based reveal in de useEffect blijven INTACT — dit bepaalt de volgorde. De CSS animatie is additioneel: zodra een lijn zichtbaar wordt, animeert hij.
- `pointer-events-none` op de HTML overlay blijft staan.
- Zorg dat de animatie NIET opnieuw afspeelt bij re-renders — de key prop op elk blok (`key={blockIdx}`) zorgt hier al voor als de blokken stabiel zijn.
