**Step A: Create `aruba-leren/src/lib/tutoring/response-analyzer.ts`**

Create a pure function `analyzeKokoResponse(text: string)` that returns `{ wasCorrect: boolean | null; wasHint: boolean }`.

Detection logic for `wasCorrect = true` (child answered correctly):
- Praise patterns in Dutch: "Goed zo", "Precies!", "Heel goed", "Klopt!", "Super!", "Uitstekend", "Correct!", "Dat klopt", "Helemaal goed", "Fantastisch", "Prima!", "Knap!", "Je hebt gelijk"
- Praise patterns in Papiamento: "Bon hasi!", "Mashá bon!", "Ekselente!", "Korecto!"
- Praise patterns in Spanish: "Muy bien!", "Excelente!", "Correcto!", "Perfecto!"
- Match case-insensitive. Use a single regex with alternation.

Detection logic for `wasCorrect = false` (child answered incorrectly):
- Correction patterns: "Niet helemaal", "Bijna!", "Probeer het nog", "Dat is niet", "Helaas", "Nog niet goed", "Kijk nog eens", "Dat klopt niet"
- Also Papiamento: "Awor purba", "No ta korecto"
- Also Spanish: "No es correcto", "Intenta de nuevo"
- Match case-insensitive.

Detection logic for `wasHint = true` (Koko gave a hint):
- Hint patterns: "Hint:", "Een hint", "Ik geef je een tip", "Laten we het anders", "Denk eens aan", "Wat als je", "Probeer eens", "Een aanwijzing", "Tip:", "Stel je voor"
- Also detect Socratic redirect patterns: "Wat denk jij?", "Hoe zou je", "Kun je uitleggen", "Wat als"
- Match case-insensitive.

If neither correct nor incorrect patterns match, return `wasCorrect: null` (indeterminate — e.g., Koko asking a new question, greeting, or explaining). When `wasCorrect` is null, do NOT call recordAnswer — only call it for definitive correct/incorrect signals.

Export the interface:
```typescript
export interface ResponseAnalysis {
  wasCorrect: boolean | null;
  wasHint: boolean;
}
export function analyzeKokoResponse(text: string): ResponseAnalysis
```

**Step B: Wire into chat route `onFinish` callback**

In `aruba-leren/src/app/[locale]/api/tutor/chat/route.ts`:

1. Add imports at top:


2. In the `onFinish: async ({ text, usage }) => { ... }` callback, AFTER the existing `saveMessage()` and `recordTokenUsage()` calls, add the response analysis and counter update logic:

   ```typescript
   // Analyze Koko's response for correctness and hint signals
   const analysis = analyzeKokoResponse(text);

   // Build metadata updates starting from existing session metadata
   const metadataUpdates: Partial<SessionMetadata> = {
     total_messages: (currentSession.metadata.total_messages || 0) + 2,
     tokens_used: (currentSession.metadata.tokens_used || 0) + totalTokens,
   };

   // Record answer if we detected a definitive correct/incorrect signal
   if (analysis.wasCorrect !== null) {
     const answerUpdates = recordAnswer(currentSession, analysis.wasCorrect);
     Object.assign(metadataUpdates, answerUpdates);

     // Update local session reference so adjustDifficulty reads fresh counters on next request
     Object.assign(currentSession.metadata, answerUpdates);
   }

   // Increment total_hints_given when Koko gives hints
   if (analysis.wasHint) {
     metadataUpdates.total_hints_given = (currentSession.metadata.total_hints_given || 0) + 1;
     currentSession.metadata.total_hints_given = metadataUpdates.total_hints_given;
   }

   // Persist all metadata updates in a single call
   await updateSessionMetadata(currentSession.id, metadataUpdates);
   ```

3. REMOVE the existing separate `updateSessionMetadata` call at the end of onFinish (lines ~236-239 that only update total_messages and tokens_used) since we now do it in the combined call above. There should be exactly ONE `updateSessionMetadata` call inside onFinish.

4. Also save `was_correct` and `hints_given` in the message metadata for auditability. Update the `saveMessage` call for the assistant message:
   ```typescript
   await saveMessage(currentSession.id, 'assistant', text, {
     tokens_used: totalTokens,
     was_correct: analysis.wasCorrect,
     hints_given: analysis.wasHint ? 1 : 0,
     difficulty_at_time: currentSession.difficulty_level,
   });
   ```
   Note: Move the `analyzeKokoResponse(text)` call BEFORE `saveMessage` so the analysis result is available for the message metadata. The order inside onFinish should be:
   1. `analyzeKokoResponse(text)`
   2. `saveMessage(...)` with analysis in metadata
   3. `recordTokenUsage(...)`
   4. Build combined `metadataUpdates` with answer + hint counters + message count + tokens
   5. Single `updateSessionMetadata(...)` call

**Important:** The `onFinish` callback captures `currentSession` via closure. The `recordAnswer()` function is pure — it reads `currentSession.metadata` and returns a partial update. We merge that update into the metadata before persisting. This means the NEXT request will read the fresh counters from Supabase, and `adjustDifficulty()` (already called at the top of the route) will see the incremented values and trigger difficulty changes when thresholds are met.
