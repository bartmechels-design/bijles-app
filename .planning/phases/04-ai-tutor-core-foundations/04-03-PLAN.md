**2. Create `aruba-leren/src/components/tutor/ChatInterface.tsx`** — Client Component ("use client"):

This is the main chat component using Vercel AI SDK's `useChat` hook.

```typescript
'use client';

import { useChat } from 'ai/react';
```

**useChat configuration:**
```typescript
const { messages, input, handleInputChange, handleSubmit, isLoading, error } = useChat({
  api: `/${locale}/api/tutor/chat`,
  body: {
    sessionId: currentSessionId,
    subject,
    childId,
  },
  initialMessages: existingSessionId ? [] : [{
    id: 'koko-greeting',
    role: 'assistant',
    content: t('tutor.kokoGreeting', { childName, subject: subjectLabel }),
  }],
  onError: (err) => {
    // Handle rate limiting (429) and other errors
    if (err.message.includes('429')) {
      setRateLimited(true);
    }
  },
});
```

**UI layout:**
- Fixed-height chat area with scroll (flex-col, overflow-y-auto, max-h calculated)
- Messages rendered via `<ChatMessage>` component
- Auto-scroll to bottom on new messages (useEffect with ref)
- Input area at bottom:
  - Text input (not textarea — kids type short responses)
  - Send button with arrow icon
  - Disabled while `isLoading` (show "Koko denkt na..." state)
- If `isLoading`, show typing indicator (3 animated dots or Koko thinking animation)
- If `error`, show friendly error message from i18n
- If `rateLimited`, show rate limit message and disable input
- SessionTimer component in header area

**Session management in client:**
- On mount, if `existingSessionId` is provided, fetch existing messages via API or pass as prop
- If no existingSessionId, the first message exchange will create a session (the API route handles this)
- Store current sessionId in state (set after first API response via response headers or body)

**Styling:**
- Kid-friendly: large text (text-lg), rounded bubbles, colorful
- Koko messages: sky-blue background, left-aligned, with small monkey emoji prefix
- Child messages: amber background, right-aligned
- Input: large, rounded, with prominent send button
- Mobile-first: works well on phone screens
