**SQL schema (include as top-of-file comment AND as a separate SQL section for the executor to run):**

```sql
-- Run this in Supabase SQL Editor
CREATE TABLE IF NOT EXISTS tutoring_sessions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  child_id UUID NOT NULL REFERENCES children(id) ON DELETE CASCADE,
  subject TEXT NOT NULL CHECK (subject IN ('taal', 'rekenen', 'begrijpend_lezen', 'geschiedenis', 'aardrijkskunde', 'kennis_der_natuur')),
  difficulty_level INT NOT NULL DEFAULT 1 CHECK (difficulty_level BETWEEN 1 AND 5),
  started_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  last_activity_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  ended_at TIMESTAMPTZ,
  metadata JSONB NOT NULL DEFAULT '{"consecutive_correct": 0, "consecutive_incorrect": 0, "total_hints_given": 0, "total_messages": 0, "tokens_used": 0, "igdi_phase": "instructie"}'::jsonb,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE TABLE IF NOT EXISTS tutoring_messages (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  session_id UUID NOT NULL REFERENCES tutoring_sessions(id) ON DELETE CASCADE,
  role TEXT NOT NULL CHECK (role IN ('user', 'assistant')),
  content TEXT NOT NULL,
  metadata JSONB,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- Indexes for performance
CREATE INDEX IF NOT EXISTS idx_tutoring_sessions_child_id ON tutoring_sessions(child_id);
CREATE INDEX IF NOT EXISTS idx_tutoring_sessions_child_subject ON tutoring_sessions(child_id, subject);
CREATE INDEX IF NOT EXISTS idx_tutoring_messages_session_id ON tutoring_messages(session_id);
CREATE INDEX IF NOT EXISTS idx_tutoring_messages_session_created ON tutoring_messages(session_id, created_at);

-- RLS policies
ALTER TABLE tutoring_sessions ENABLE ROW LEVEL SECURITY;
ALTER TABLE tutoring_messages ENABLE ROW LEVEL SECURITY;

-- Parents can view/create sessions for their own children
CREATE POLICY "Parents can manage their children's sessions"
  ON tutoring_sessions
  FOR ALL
  USING (
    child_id IN (
      SELECT c.id FROM children c
      JOIN profiles p ON c.parent_id = p.id
      WHERE p.user_id = auth.uid()
    )
  );

-- Parents can view/create messages for their children's sessions
CREATE POLICY "Parents can manage their children's messages"
  ON tutoring_messages
  FOR ALL
  USING (
    session_id IN (
      SELECT ts.id FROM tutoring_sessions ts
      JOIN children c ON ts.child_id = c.id
      JOIN profiles p ON c.parent_id = p.id
      WHERE p.user_id = auth.uid()
    )
  );
```

**Exported functions:**

- `createSession(childId: string, subject: Subject): Promise<TutoringSession | null>` — Insert new session, return it. Validate child_id exists via query before inserting.
- `getSession(sessionId: string): Promise<TutoringSession | null>` — Fetch session by ID.
- `getActiveSession(childId: string, subject: Subject): Promise<TutoringSession | null>` — Find most recent session for child+subject that hasn't ended (ended_at IS NULL) and was active within last 30 minutes (last_activity_at > now - 30min). Returns null if no active session found.
- `saveMessage(sessionId: string, role: 'user' | 'assistant', content: string, metadata?: MessageMetadata): Promise<void>` — Insert message and update session's last_activity_at + metadata.total_messages.
- `getRecentMessages(sessionId: string, limit?: number): Promise<TutoringMessage[]>` — Fetch last N messages (default 15, per MAX_CONTEXT_MESSAGES) ordered by created_at ascending. This is used to build the context window for the AI.
- `updateSessionMetadata(sessionId: string, metadata: Partial<SessionMetadata>): Promise<void>` — Update session metadata (difficulty, IGDI phase, performance counters).
- `endSession(sessionId: string): Promise<void>` — Set ended_at to now.

**2. Create `aruba-leren/src/lib/tutoring/difficulty-adjuster.ts`:**

Implements the adaptive difficulty algorithm per TUTOR-06:

- `adjustDifficulty(session: TutoringSession): DifficultyAdjustment` — Pure function that examines session metadata:
  - If `consecutive_correct >= 3` AND `difficulty_level < 5` → increase by 1, reason: 'consecutive_correct'
  - If `consecutive_incorrect >= 3` AND `difficulty_level > 1` → decrease by 1, reason: 'needs_easier'
  - Also decrease if `total_hints_given` for current problem exceeds 3
  - Otherwise → no change, reason: 'no_change'
  - Return `DifficultyAdjustment` with instruction string in Dutch for the AI prompt

- `recordAnswer(session: TutoringSession, wasCorrect: boolean): Partial<SessionMetadata>` — Returns updated metadata fields:
  - If correct: increment consecutive_correct, reset consecutive_incorrect
  - If incorrect: increment consecutive_incorrect, reset consecutive_correct
  - Returns the partial metadata to merge (caller does the Supabase update)

Create the main streaming chat endpoint at `aruba-leren/src/app/[locale]/api/tutor/chat/route.ts`.

**IMPORTANT:** This is a Next.js App Router route handler (not a server action). It uses the Vercel AI SDK's `streamText` to stream responses.

**POST handler flow:**

1. **Parse request body:** Expect JSON with `{ messages: Array<{role, content}>, sessionId: string, subject: Subject, childId: string }`. Validate with basic checks (not zod schema — keep it simple).

2. **Authenticate:** Use `createClient()` from supabase/server. Call `supabase.auth.getUser()`. If not authenticated, return 401.

3. **Verify ownership:** Query profiles table for user_id, then verify the childId belongs to this parent (children.parent_id = profiles.id). If not owned, return 403.

4. **Get child info:** Fetch child record (first_name, age, grade) for prompt context.

5. **Rate limit check:** Call `checkTokenBudget(childId)`. If not allowed, return 429 with JSON `{ error: "Dagelijks limiet bereikt", remaining: 0 }`.

6. **Get or create session:** If sessionId provided, call `getSession(sessionId)` and verify it belongs to this child. If no sessionId, call `getActiveSession(childId, subject)` to try resuming, or `createSession(childId, subject)` to start fresh.

7. **Load context messages:** Call `getRecentMessages(sessionId, MAX_CONTEXT_MESSAGES)` to get conversation history. Merge with any new messages from the request (the latest user message).

8. **Save user message:** Call `saveMessage(sessionId, 'user', latestUserMessage.content)`.

9. **Check difficulty adjustment:** Call `adjustDifficulty(session)`. If adjustment needed, include the instruction in the system prompt.

10. **Build system prompt:** Call `buildSystemPrompt(subject, childLanguage, childAge, childName, difficultyLevel, igdiPhase)` using session and child data. The child's preferred language defaults to the locale from the URL path (extract from request URL).

11. **Stream response:** Use Vercel AI SDK:
```typescript
import { streamText } from 'ai';

const result = streamText({
  model: TUTOR_MODEL,
  system: systemPrompt,
  messages: contextMessages,  // Last 15 messages + new user message
  maxTokens: MAX_TOKENS_PER_RESPONSE,
  temperature: TEMPERATURE,
  onFinish: async ({ text, usage }) => {
    // Save assistant message with token usage metadata
    await saveMessage(sessionId, 'assistant', text, {
      tokens_used: usage.totalTokens,
    });
    // Record token usage for rate limiting
    await recordTokenUsage(sessionId, usage.totalTokens);
    // Update session last_activity_at
    await updateSessionMetadata(sessionId, {
      total_messages: (session.metadata.total_messages || 0) + 2,  // user + assistant
      tokens_used: (session.metadata.tokens_used || 0) + usage.totalTokens,
    });
  },
});

return result.toDataStreamResponse();
```

12. **Error handling:** Wrap entire handler in try/catch. On Anthropic API errors, return 502 with generic message (don't expose API details). On other errors, return 500.
