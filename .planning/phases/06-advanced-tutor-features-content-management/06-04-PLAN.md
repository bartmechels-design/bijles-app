<objective>
Add Huiswerk Hulp mode to the tutoring session. When a child attaches an image before starting (indicating they want help with homework), Koko switches to a dedicated huiswerk prompt: analyze the assignment, identify errors, guide the child Socratically toward correct answers without giving them directly.

Purpose: Children often need targeted help with specific homework assignments rather than general subject practice. The image upload flow already exists in ChatInterface — this plan extends it with automatic mode detection and a specialized prompt.

Approach: huiswerkMode stored as JSONB flag in existing tutoring_sessions.metadata (no DB schema change needed — avoids the CHECK constraint on session_type). ChatInterface detects image-before-session, sets the flag, and the chat route reads metadata.huiswerkMode to select the right prompt builder.

Output: buildHuiswerkPrompt() in system-prompts.ts, huiswerk detection in route.ts, banner + auto-send in ChatInterface.
</objective>

```typescript
export function buildHuiswerkPrompt(
  subject: Subject,
  language: TutoringLanguage,
  childAge: number,
  childName: string,
  childGrade: number
): string {
  const staticPrompt = KOKO_BASE_PROMPT + '\n\n' + SOCRATIC_GUARD_PROMPT;
  const subjectPrompt = SUBJECT_PROMPTS[subject];
  const languageContext = buildLanguageContext(language, childAge);

# Deze Leerling

**Naam**: ${childName}
**Leeftijd**: ${childAge} jaar${gradeInfo}

${childName} heeft huiswerk meegenomen en wil hier hulp bij.
`;

  const huiswerkInstructions = `
# SPECIALE MODUS: HUISWERK HULP

${childName} heeft een foto van huiswerk geüpload. Dit is jouw werkwijze:

## Stap 1: Analyseer (doe dit DIRECT bij het eerste bericht)
- Bekijk de afbeelding nauwkeurig: welk vak, welke opgaven, wat is al ingevuld?
- Bepaal het niveau: past dit bij klas ${childGrade}?
- Identificeer patronen: maakt ${childName} steeds dezelfde fout? Mist er een basisregel?

## Stap 2: Begin met het positieve
- Noem altijd eerst wat al goed is: "Ik zie dat je opgave 1 en 2 al helemaal goed hebt gedaan!"
- Ga dan pas naar de opgaven die aandacht nodig hebben

## Stap 3: Begeleid Socratisch (NOOIT het antwoord geven)
- Kies de EERSTE foutieve of openstaande opgave
- Gebruik de Socratische methode: stel vragen die leiden naar het juiste inzicht
- Eén opgave tegelijk — ga pas door als de huidige opgave begrepen is
- Bij een patroonfout: leg de REGEL uit via vragen, dan terug naar de oefening

## Stap 4: Huiswerk afwerken
- Ga systematisch door alle opgaven
- Eindig met een samenvatting: "Vandaag hebben we geleerd dat..."
- Geef een korte tip mee voor thuis

## Absolute regels
- Geef NOOIT het antwoord direct — ook niet als ${childName} erom vraagt
- Blijf ALTIJD binnen het vak dat zichtbaar is in het huiswerk
- Als de afbeelding onduidelijk is, vraag dan om een duidelijkere foto
- Reageer ALTIJD in de instructietaal van deze sessie

## Eerste reactie
Begin DIRECT met de analyse van het huiswerk. Geen lange intro — ga meteen aan de slag.
`;

  return [

    subjectPrompt,
```

   The existing guard is:

   ```typescript
   if ((!input.trim() && !pendingImage) || isLoading) return;
   ```
   This already allows submission with pendingImage — so the auto-send is triggered differently.

   Instead, add a `useEffect` that watches `huiswerkMode` and `pendingImage`:
   ```typescript
   // Auto-send first message when huiswerk mode activates with a pending image
   useEffect(() => {
     if (huiswerkMode && pendingImage && !sessionStarted && !isLoading) {
       // Trigger submit after a short delay so state has settled
       const timer = setTimeout(() => {
         // Programmatically submit: create a synthetic form event
         handleSubmit({ preventDefault: () => {} } as React.FormEvent);
       }, 600);
       return () => clearTimeout(timer);
     }
   }, [huiswerkMode, pendingImage]); // eslint-disable-line react-hooks/exhaustive-deps
   ```

   Note: The auto-submit fires once, then `sessionStarted` becomes true and `pendingImage` clears, so the effect won't re-trigger.
